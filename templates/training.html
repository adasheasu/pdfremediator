üìö Welcome to PDF Remediation Training

This training will teach you how to manually remediate PDFs for accessibility using Adobe Acrobat. While our automated tool handles many issues, understanding manual remediation helps you:

  • Handle complex documents that need expert review
  • Fix specialized issues the automated tool can't address
  • Understand accessibility principles deeply
  • Verify and test automated remediation results
  • Become an accessibility expert for your team
üìÑ Practice Document: Follow along with this training using our sample PDF document. This document contains the same accessibility issues shown in the training screenshots, so you can practice each step hands-on.
Estimated Time: 30-45 minutes | Difficulty: Intermediate | Software Required: Adobe Acrobat Pro DC
1

Open PDF and Find Accessibility Checker

Let's begin by opening your PDF and locating the Accessibility Checker tool.

  1. Open your PDF in Adobe Acrobat Pro DC
  2. Find the Accessibility Checker using one of these methods:
    • Search method: Use the search tool in the upper right (Command+F or Ctrl+F) and search for "accessibility checker"
    • Menu method: Navigate to Tools ‚Üí Prepare for Accessibility ‚Üí Accessibility Checker
Finding the Accessibility Checker in Adobe Acrobat Pro DC
Pro Tip: Consider creating a duplicate of your document before starting remediation. This gives you an original backup in case the remediation process doesn't go as planned.
2

Run the Accessibility Checker

Now we'll run the checker to identify all accessibility issues in the document.

  1. Once you've found the Accessibility Checker, click "Start Checking"
  2. Wait for the tool to analyze your document (this usually takes a few seconds)
Click "Start Checking" to begin the accessibility analysis
What Happens Next: The accessibility checker will scan your entire PDF and generate a report showing all issues that need to be fixed. Don't worry if you see many issues - we'll address them systematically!
3

View the Accessibility Report

The report will appear showing accessibility issues organized in categories.

When the report first appears, all the issue categories are collapsed (closed). You'll see bolded field names with disclosure triangles next to them.

Accessibility Report with closed dropdown menus
Understanding the Report Colors:
  • ‚úñ Red (Failed): Critical issues that must be fixed for accessibility
  • ‚ö† Yellow (Needs Manual Check): Items requiring human review and judgment
  • ‚úì Green (Passed): Requirements that are already met
4

Open the Dropdown Menus

Expand each bolded field to see the specific issues within each category.

  1. Click on each bolded field name to expand it into a dropdown menu
  2. Review the specific issues listed under each category
  3. Don't be overwhelmed! We'll fix them one by one
Accessibility Report with opened dropdown menus showing all issues
You're Ready to Start Fixing! Now that you can see all the issues, we'll systematically work through them. Most issues can be fixed with just a few clicks.
5

Fix "Tagged PDF - Failed"

This is the most critical fix - adding tags provides the structure that screen readers need to navigate your document.

  1. Scroll to the top of the accessibility issues list
  2. Look for "Tagged PDF - Failed"
  3. Right-click on "Tagged PDF - Failed"
  4. Select "Fix" from the dropdown menu
  5. Acrobat will automatically add tags to your document
Select "Fix" to automatically add tags to the PDF
üéâ Major Progress! Once you fix the tagging, you'll notice that a majority of the accessibility issues are automatically resolved! Tags provide the document structure that makes content accessible to screen readers.
Save Your Work: Consider saving your document frequently to ensure you don't lose your remediation work. Use Command+S (Mac) or Ctrl+S (Windows) after each major fix.
6

Find Missing Alternate Text

Alt text describes images for screen reader users. Let's identify which images need descriptions.

  1. In the Accessibility Report, look for "Alternate Text" section
  2. Expand it by clicking on the disclosure triangle
  3. Open "Figures alternate Text - Failed"
  4. You'll see a list of figures (images) that need alt text
Finding missing alternate text in the accessibility report
Why Alt Text Matters: Screen readers announce images using alt text. Without it, blind users miss important visual information. Good alt text is concise but descriptive.
7

Start Adding Alt Text

We'll add alt text to each image that needs it.

  1. In the issues list, find "Figure 1" (or the first figure listed)
  2. Right-click on "Figure 1"
  3. Select "Fix" from the dropdown menu
Select "Fix" to open the alt text editor

What Happens Next: A pop-up dialog will appear that lets you add or edit the alternate text for this image.

8

View the Alt Text Dialog

The alt text editor shows you which image you're describing.

When the alt text dialog opens, you'll notice:

  • The image in question is highlighted in the document
  • A text field where you can enter the description
  • A checkbox to mark the image as decorative (if it's purely visual)
Alt text popup with highlighted image (in this case, the Learning Enterprise logo)
Identifying the Image: The highlighting helps you confirm which image you're describing. In this example, it's the Learning Enterprise logo at the bottom right side of the document.
9

Write Alt Text or Mark Decorative

Now you'll add a description or mark the image as decorative.

  1. In the alt text field, type a concise description of the image
  2. OR if the image is purely decorative (adds no meaningful content), check "Decorative Figure"
  3. Click "Save & Close"
Writing alt text and clicking Save & Close
Writing Good Alt Text:
  • Be concise: Aim for one sentence, usually under 125 characters
  • Be descriptive: Describe what's important about the image in context
  • Don't start with "Image of...": Screen readers already announce it's an image
  • Consider context: Is this image important for understanding the content? If yes, describe it. If it's purely decorative, mark it as such.
When to Mark as Decorative: Use "Decorative Figure" for images that are purely visual (borders, dividers, design elements) or when the content is fully described in surrounding text. Example: A photo of a document next to text that fully explains the document's content.
10

Verify Alt Text Warning Is Gone

Check that the alt text issue has been resolved for this image.

After saving the alt text:

  • The warning for that specific figure disappears from the accessibility report
  • If there are more images needing alt text, repeat steps 7-9 for each one
  • Continue until all "Figures alternate Text - Failed" issues are resolved
Alt text warning is now gone from the report
Great Work! You've made your images accessible to screen reader users. Next, we'll tackle table accessibility.
11

Open the Reading Order Tool

Tables need proper headers and summaries for screen readers. Let's start by opening the Reading Order tool.

  1. In the accessibility menu, find and click "Fix reading order"
  2. Or navigate to Tools ‚Üí Accessibility ‚Üí Reading Order
Opening the Reading Order tool from accessibility options
What Is Reading Order? The Reading Order tool lets you see how content is structured and tagged in your PDF. It's essential for fixing tables and ensuring content is read in the correct sequence by screen readers.
12

View the Reading Order Panel

The Reading Order tool will open as a panel showing tags on your document elements.

You'll see:

  • A panel on the right side of the screen
  • Numbered boxes overlaid on your document showing the reading order
  • Different colors indicating different element types (text, images, tables, etc.)
Reading Order panel displays with tags visible on document elements
Understanding the Tags: Each numbered box represents a content element. The numbers show the order in which screen readers will announce the content.
13

Select the Table Tag

Now we'll select the table that needs fixing.

  1. Look for the numbered tag box that covers your table
  2. Click on that tag box - the table will be highlighted in blue
  3. Right-click on the highlighted table
  4. A context menu will appear with table editing options
Table selected and highlighted, with context menu open
14

Add Table Summary

A table summary provides context about what the table contains and its purpose.

  1. From the context menu, select "Edit Table Summary"
  2. A dialog box will open
Select "Edit Table Summary" from the menu
Why Table Summaries Matter: Screen readers announce the table summary before reading the table contents. This helps users understand the table's purpose and decide if they want to navigate through it. It's like a caption that explains what the table is about.
15

Write the Table Summary

Write a brief description of what the table contains.

  1. In the Table Summary dialog, type a concise description
  2. Describe what information the table presents and its organization
  3. Click "OK" to save
Enter a descriptive table summary and click OK
Writing Good Table Summaries:
  • Be concise: One or two sentences usually suffice
  • Describe the data: What information does the table present?
  • Note organization: How is the data arranged? (by rows, columns, categories)
  • Example: "Course schedule showing class times, locations, and instructors organized by day of the week"
16

Open Table Editor to Fix Headers

Now we'll define which cells are headers so screen readers can announce them correctly.

  1. Right-click on the table again (the same way you did before)
  2. From the context menu, select "Table Editor"
  3. The table editor mode will activate
Select "Table Editor" to begin defining table headers
Why Table Headers Matter: When screen readers announce table data, they also announce the relevant headers for each cell. This provides context for what each data point represents. Without proper headers, users hear disconnected data without understanding what it means.
17

Table Editor Mode Activated

When Table Editor mode is active, the table will be highlighted in pink.

Visual indicators:

  • The entire table is now highlighted in pink/magenta
  • Individual cells are clearly outlined
  • You can now right-click on individual cells to set their properties
Table highlighted in pink - ready to edit header properties
You're Ready! The pink highlighting indicates you can now edit table cell properties. Next, we'll define which cells are headers.
18

Open Cell Properties

We'll start by defining the first header cell.

  1. Find the top-left cell of your table (usually the first header)
  2. Right-click on that cell
  3. Select "Table Cell Properties" from the menu
Right-click on a cell and select "Table Cell Properties"

What's Next: The Table Cell Properties dialog will open, allowing you to designate this cell as a header.

19

Set Header Cell Properties

Now we'll designate this cell as a header and specify what it labels.

  1. In the Properties dialog, select "Header Cell"
  2. Choose the appropriate scope:
    • "Column" - if this header labels the cells below it
    • "Row" - if this header labels the cells to the right of it
    • "Both" - if this header labels both column and row
    • "None" - if it's not actually a header
  3. Adjust row and column span if the header covers multiple cells
  4. Click "OK"
Set the cell as a Header Cell and choose the appropriate scope
Understanding Scope:
  • Column headers typically appear in the top row and describe what's in each column below
  • Row headers typically appear in the left column and describe what's in each row to the right
  • Some tables have both column and row headers
Repeat for All Headers: After setting this cell, continue the same process for all cells that act as headers in your table. It's recommended to decide which cells are headers based on how the table organizes information for the reader.
Almost Done with Tables! Once you've set all header cells, your table will be fully accessible. Screen readers will now announce headers along with data, providing context for every cell.

üåê Section 5: Set Language & Recheck

Setting the correct document language is crucial for screen readers to pronounce text correctly. After fixing language settings, we'll run the checker again to ensure all issues are resolved.

20

Check Primary Language

The document's language setting tells screen readers how to pronounce the text.

In the Accessibility Checker report:

  1. Right-click on "Primary Language"
  2. Click "Fix" from the context menu
Fixing the Primary Language setting
Why This Matters: If the language is not set or set incorrectly, screen readers will pronounce words using the wrong language rules, making content incomprehensible to users. For example, Spanish text read with English pronunciation rules sounds completely wrong.
If your document contains multiple languages (e.g., English with Spanish quotes), you can set language tags for specific sections later. The Primary Language sets the default for the entire document.
21

Set Language to English

Select the correct language for your document.

In the Reading Options dialog:

  1. Make sure the Language dropdown is set to "English (United States)" (or the primary language of your document)
  2. Click "OK" to save the setting
Setting the document language to English
Common language options include English (United States), English (United Kingdom), Spanish (Mexico), French, German, and many others. Choose the variant that matches your document's dialect for best results.
22

Run the Accessibility Checker Again

After making fixes, always re-run the checker to catch any remaining issues.

  1. Click the "Check for Accessibility" button again
  2. Review the updated report to see what issues remain
  3. Verify that "Primary Language" is now marked as passed
Re-running the checker to verify fixes
Progress Check: At this point, you should see checkmarks for:
  • ‚úì Tagged PDF
  • ‚úì All images have alternate text
  • ‚úì All tables have headers and summaries
  • ‚úì Primary Language is set
The Accessibility Checker doesn't catch everything automatically. Some issues require manual review, like verifying that heading levels are used correctly or that alt text is actually descriptive (not just present).
23

Open the Tags Panel

The Tags panel shows the document's structure. We need to check for proper heading hierarchy.

  1. Look for the "Tags" icon in the left sidebar (looks like a tag or label icon)
  2. Click on the "Tags" icon to open the Tags panel
  3. You'll see the document's tag tree structure
Opening the Tags panel to view document structure
What to Look For: A properly structured document should have heading tags (H1, H2, H3, etc.) in the tag tree. If you see only paragraph tags (<P>) without any heading tags (<H1>, <H2>, etc.), you'll need to manually add them in the next section.
The tag tree is hierarchical - just like an outline. Headings should be nested properly: H1 for main title, H2 for major sections, H3 for subsections, etc. Never skip levels (e.g., don't go from H1 directly to H3).

üìë Section 6: Add Headings & Test

Headings provide structure and navigation for screen reader users. We'll add proper heading tags to your document and test the final result to ensure everything works correctly.

24

Check Tags for Headings

Examine the tag structure to see if proper heading tags exist.

In the Tags panel:

  1. Expand the tag tree by clicking the arrows next to tags
  2. Look for heading tags: <H1>, <H2>, <H3>, etc.
  3. If you only see <P> (paragraph) tags where there should be headings, you'll need to add them manually
Checking the tag structure for headings
Heading Hierarchy Rules:
  • H1: Main document title (use only once)
  • H2: Major sections
  • H3: Subsections under H2
  • H4-H6: Further nested subsections
Never skip levels! Don't go from H1 to H3 without an H2 in between.
Screen reader users rely heavily on headings to navigate documents. Many users jump between headings rather than reading linearly. Missing or improperly structured headings make navigation extremely difficult.
25

Open Reading Order Panel

Use the Reading Order tool to identify and tag headings in your document.

  1. Go to Tools ‚Üí Accessibility ‚Üí Reading Order
  2. Or search for "Reading Order" in the Tools search bar
  3. Click to open the Reading Order panel
Opening the Reading Order panel
The Reading Order tool is essential for fixing document structure. You'll use it to designate which text should be headings, which should be regular text, and in what order content should be read.
26

Select Text to Tag as Heading

Identify the text that should be marked as a heading and select it.

With the Reading Order panel open:

  1. Click and drag to draw a box around the text that should be a heading (usually the document title or main heading)
  2. Or click on an existing text element to select it
  3. The selected area will be highlighted
Selecting text to tag as a heading
How to Identify Headings:
  • Look for text that's larger, bolder, or visually distinct from body text
  • Document titles should be H1
  • Section titles should be H2
  • Subsection titles should be H3
Be precise with your selection boxes. Try to select only the heading text, not surrounding paragraphs or whitespace. You can always adjust tags later in the Tags panel if needed.
27

Apply the Heading Tag

Tag the selected text as a heading using the Reading Order panel buttons.

  1. In the Reading Order panel, click the "Heading 1" button to tag the main title
  2. Or click "Heading 2", "Heading 3", etc., depending on the heading level
  3. The tag will be applied and you'll see the structure update in the Tags panel
Applying a Heading 1 tag to the selected text
Repeat for All Headings: Continue this process for all headings in your document:
  • Select the text
  • Click the appropriate heading level button
  • Work from top to bottom through the document
  • Maintain proper hierarchy (H1 ‚Üí H2 ‚Üí H3, never skip levels)
After tagging headings, check the Tags panel to verify the structure. You should see a logical hierarchy with H1 at the top, followed by H2s for sections, H3s for subsections, etc.
28

Test with Screen Reader / Verify Reading Order

The final and most important step: test your document to ensure it's truly accessible.

Testing options:

  1. View Reading Order: In the Reading Order panel, verify the numbered order makes sense
  2. Read Out Loud: Go to View ‚Üí Read Out Loud ‚Üí Activate Read Out Loud, then navigate with keyboard shortcuts to hear how screen readers will interpret your document
  3. Test with Actual Screen Reader:
    • Windows: Use NVDA (free) or JAWS
    • Mac: Use built-in VoiceOver (Cmd+F5)
    • Navigate by headings to test structure
    • Verify all images have alt text read aloud
    • Check that table headers are announced
Testing the document with Read Out Loud and viewing reading order
What to Test For:
  • ‚úì Content reads in logical order (not jumping around)
  • ‚úì Headings are announced with correct levels (H1, H2, H3)
  • ‚úì Images have alt text that's read aloud
  • ‚úì Table headers are announced when navigating cells
  • ‚úì No content is skipped or unreadable
  • ‚úì Language pronunciation is correct
Screen Reader Keyboard Shortcuts (NVDA/JAWS):
  • H key: Jump to next heading
  • G key: Jump to next graphic/image
  • T key: Jump to next table
  • Arrow keys: Read line by line
  • Insert+Down arrow: Read all from current position

  • ‚úì Save your work frequently
  • ‚úì Run the Accessibility Checker one final time
  • ‚úì Keep the original PDF as a backup
  • ‚úì Test with real screen reader users when possible
